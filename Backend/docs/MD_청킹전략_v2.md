# 청약제도안내(MD) 청킹 전략 v2 — 표 → 자연어 완전 변환

## 1. 배경 — 문제 발견

`guide_chunks`(청약Home 청약제도안내, 72개)는 다른 5개 collection(law/faq/manual/lh_guide/web_faq) 대비 임베딩 거리값이 체계적으로 높게 나왔다.

| Collection | min_distance 평균 |
|---|---|
| law_chunks | 0.5345 |
| faq_chunks | 0.4540 |
| manual_chunks | 0.4987 |
| lh_guide_chunks | 0.4940 |
| web_faq_chunks | 0.4691 |
| **guide_chunks** | **1.0040** |

### 원인 진단

`guide_chunks`의 self-distance(청크 본문 일부를 그대로 질문으로 사용했을 때의 거리)는 평균 0.21로 충분히 낮음 → 임베딩 모델/청크 자체의 구조적 문제는 아님.

반면 실제 질문(자연어 문장)과의 거리는 평균 1.06으로 매우 높음 → **"질문 형태 ↔ 문서 형태의 불일치"** 가 원인.

`guide_chunks`의 청크 본문은 마크다운 표(`|`, `---`, `**`) 구조가 그대로 남아 있어, "신혼부부 특별공급 소득 기준 130%" 같은 자연어 질문과 형태적으로 크게 달랐음. 이로 인해 라우팅에서 `guide_chunks`를 우대해도 거리값 차이로 인해 실질적으로 검색 결과에 거의 포함되지 못함.

### 해결 방향

표를 자연어 질의-친화적 문장으로 **완전히 대체**하여 문서 형태를 질문 형태와 가깝게 만든다 (원본 표는 보존하지 않음).

## 2. 표 → 자연어 변환 (`convert_tables_to_text.py`)

원본 `청약제도안내.md`의 모든 마크다운 표(39개)를 행 단위 자연어 문장으로 변환. 표 구조에 따라 3가지 규칙을 적용한다.

### 규칙 1 — 가구원수별 금액표 (소득기준표)

헤더의 마지막 N개 컬럼이 `3인 이하` / `4인` / `5인` 같은 가구원수 형태인 표.

```
변환 전:
| 공급유형 | 소득 구분 | 기준 | 3인 이하 | 4인 | 5인 |
| 신생아 우선공급 (기준소득, 25%) | 부부 중 한명만 소득이 있는 경우 | 100% 이하 | ~7,533,763 | ~8,802,202 | ~9,326,985 |

변환 후:
신생아 우선공급 (기준소득, 25%), 소득 구분은 부부 중 한명만 소득이 있는 경우, 100% 이하인 경우,
3인 이하 ~7,533,763원, 4인 ~8,802,202원, 5인 ~9,326,985원입니다.
```

### 규칙 2 — 가점항목 표 (구간|점수 1~2세트)

헤더가 `... | 점수` 또는 `... | 점수 | ... | 점수` 형태(좌우 2세트가 한 표에 있는 경우 포함).

```
변환 전:
| 무주택기간 | 점수 | 무주택기간 | 점수 |
| 1년 미만 | 2 | 8년 이상 ~ 9년 미만 | 18 |

변환 후:
1년 미만: 2점
8년 이상 ~ 9년 미만: 18점
```

### 규칙 3 — 일반 매트릭스 표 (그 외 전부)

1열을 행 식별자로 보고, 나머지 컬럼명=값을 "은/는" 조사를 붙여 나열. 컬럼명이 `내용`/`값`/`설명`/`비고`처럼 의미 없는 라벨이면 컬럼명을 생략.

```
변환 전:
| 항목 | 내용 |
| 대상주택 | 전용면적 85m² 이하의 분양주택 |

변환 후:
대상주택: 전용면적 85m² 이하의 분양주택.
```

```
변환 전:
| 주거전용면적 | 투기과열지구 | 청약과열지역 |
| 60m² 이하 | 가점제 40% | 가점제 40% |

변환 후:
60m² 이하: 투기과열지구는 가점제 40%, 청약과열지역은 가점제 40%.
```

### 한국어 조사(은/는) 처리

컬럼명 끝 글자의 받침 유무에 따라 `은`(받침 있음) / `는`(받침 없음)을 자동 선택(`josa_eun_neun` 함수). 한글이 아닌 문자로 끝나면 `는`을 기본값으로 사용.

### 변환 결과

- 39개 표 전부 자연어 문장으로 대체, 표(`|`로 시작하는 줄) 0개
- 마크다운 헤더(`#`/`##`/`###`) 구조는 그대로 유지 → 기존 헤더 기반 청킹 그대로 적용 가능

## 3. 청킹 (`md_table_chunker.py`)

표가 모두 자연어로 변환됐으므로, 표 인식/분할 로직(`split_into_text_and_tables`, `pack_pieces`, `augment_with_table_description` 등)은 제거하고 **헤더 분할 + 문단 단위 분할**만 수행한다.

### 청킹 단계

1. **H1/H2/H3 헤더 기준 1차 분할** (`MarkdownHeaderTextSplitter`)
2. 각 청크가 `MAX_CHUNK_SIZE(1200자)` — prefix 길이를 뺀 effective 크기 — 이하면 그대로 사용
3. 초과하면 **문단(빈 줄) 단위로 분할**
   - 문단이 여전히 너무 크면(표 변환 결과처럼 한 줄 = 한 문장인 경우) **줄(`\n`) 단위로 재분할**
   - 분할된 문장들을 순서대로 묶어 effective 크기를 넘지 않는 청크로 패킹
4. 모든 청크 앞에 prefix `[Header_1 > Header_2 > Header_3]` 추가

### prefix 길이 보정

`effective_max = MAX_CHUNK_SIZE - len(prefix)` 로 계산하여, prefix를 더한 최종 청크 길이가 `MAX_CHUNK_SIZE`를 넘지 않도록 함. "그대로 사용" 분기와 "패킹" 분기 모두 이 `effective_max` 기준으로 판단한다 (한쪽만 보정하면 그대로 사용 분기에서 prefix 추가 후 초과하는 문제가 발생했음).
