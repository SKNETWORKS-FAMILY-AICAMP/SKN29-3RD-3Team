# 2024 주택청약 FAQ 청킹 전략 (v2, pypdf 기반)

## 1. 문제 배경

`pdftotext`(poppler)가 Windows 환경에서 한글 파일명/일반 파일 모두 텍스트 추출에 실패하여(`\` 한 글자만 출력되거나 빈 결과), 결국 **pypdf로 통일**하기로 결정했습니다.

그런데 v1 청커(`RE_Q_NUM = r"^(\d{1,3})$"`, 숫자 단독 줄)는 `pdftotext` 출력 기준으로 작성되어 있었고, pypdf로는 Q번호 추출이 거의 실패해 **183개**(목표 417~480개)밖에 만들어지지 못했습니다. 그 결과 `Q98`에 실제로는 다른 질문(`Q462` 부근 내용인 "생애최초 특별공급...")이 매핑되는 등 데이터 정합성 문제가 발생했습니다.

## 2. pypdf 추출 텍스트의 패턴 차이

| | pdftotext | pypdf |
|---|---|---|
| Q번호 | 숫자가 단독 줄 | **숫자 + 공백 + 질문 텍스트가 한 줄** |
| 예시 | `"14"` (다음 줄에 질문) | `"14 단신부임의 적용 기준이 되는 세대원이란?"` |

```
pypdf 실제 출력 예 (page 38):
"나. 청약신청지역 및 우선공급
1 경기도 과천시에서 공급되는 주택의 해당 주택건설지역의 범위는?
해당 주택건설지역이란 특별시ㆍ광역시ㆍ특별자치시ㆍ특별자치도...
2 해당 주택건설지역에 거주하고 있지 않다면 청약신청이 불가능한지?
..."
```

## 3. 문서 구조 분석

273페이지 PDF를 pypdf로 읽었을 때의 구조입니다.

| 페이지 (0-indexed) | 내용 |
|---|---|
| 0~1 | 표지 / 안내문 |
| 2~31 | 목차 (로마숫자 페이지번호, "Q1.", "Q50." + 점선 leader) |
| **32~272** | **본문** |

본문의 줄 단위 패턴:

```
"Ⅰ. 청약자격(공통)"     ← 챕터 헤더 (페이지 상단 반복)
"3"                      ← 페이지 번호 (단독 숫자줄)
"청약신청지역1"           ← 소제목+섹션번호 (특수 케이스, 무시)
"가. 주요내용"           ← 중분류(섹션)
"1 경기도 과천시...?"    ← Q번호+질문 (한 줄에 붙음)
"해당 주택건설지역..."    ← 답변 (여러 줄)
"2 해당 주택건설지역...?" ← 다음 Q
```

특수 케이스: 챕터 제목과 로마숫자가 두 줄로 분리되는 경우가 있습니다.

```
"거주의무"
"Ⅶ"
```

## 4. 청킹 로직

### 4-1. 전처리 (`_clean_lines`)

1. 페이지 단위로 분리(`\f` 기준)
2. **0~31페이지(표지+목차) 전부 제외** — `BODY_START_PAGE = 32`
3. 점선(`···`, `....`)이 3개 이상인 페이지는 추가로 목차로 간주해 제외 (`is_toc_page`)
4. 페이지 번호 단독 줄(`^\d{1,3}$`) 제거 — Q번호 패턴과 충돌 방지
5. 머리말(`2024 주택청약FAQ`, `www.molit.go.kr` 등) 제거
6. **두 줄로 분리된 챕터 헤더 정규화**: `"거주의무\nⅦ"` → `"Ⅶ. 거주의무"`

### 4-2. Q번호 인식 (`RE_INLINE_Q`)

```python
RE_INLINE_Q = re.compile(r"^(\d{1,3})\s+(.+)$")
```

`"1 경기도 과천시에서..."` → `(번호="1", 질문="경기도 과천시에서...")`

다음 조건에 해당하면 Q번호가 아닌 것으로 제외합니다.

- 법령조항 (`"3. 제1호..."`, `RE_LAW_ITEM` — 숫자 뒤에 마침표가 있음)
- 챕터/섹션 헤더 (`Ⅰ.`, `가.` 등)
- 텍스트 길이가 5자 미만

### 4-3. 순증가 시퀀스 필터링 (`_filter_sequential`)

본문은 Q1~Q480까지 순서대로 증가하므로, **이전 번호보다 큰 경우만 채택**합니다. 답변 중 법령 조문이 인용되며 등장하는 `"3. 제1호..."` 같은 줄은 이 단계에서 자연스럽게 걸러집니다.

### 4-4. 챕터 경계에서 답변 절단

Q번호의 답변을 수집하던 중 **새로운 챕터 헤더(`Ⅰ.`~`Ⅹ.`)를 만나면 즉시 수집을 중단**합니다. 이게 없으면 챕터 전환 지점의 마지막 Q가 다음 챕터 도입부 텍스트까지 통째로 흡수해 비정상적으로 길어집니다.

```
[수정 전] Q462: 7,323자 (다음 챕터 "Ⅶ. 거주의무" 도입부가 통째로 붙음)
[수정 후] Q462: 174자 (정상)
```

## 6. 알려진 한계

일부 Q(예: Q371, 1,951자)는 답변 중간에 **소제목 없이 법령 원문이 길게 인용**되는 구간을 포함합니다. 이런 구간은 다음 실제 Q를 만날 때까지 이전 Q의 답변에 흡수됩니다. 다만 모두 6,000자 이내로 임베딩 토큰 한도(8,192토큰) 문제는 없어 현재는 허용 가능한 수준으로 판단했습니다.
