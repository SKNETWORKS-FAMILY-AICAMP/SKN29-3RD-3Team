
**청크 예시 (소제목 단위 분리 - 제4절 특별공급):**
```
[청크]
제4장 제4절 특별공급 > 신혼부부 특별공급

◦ 신혼부부 특별공급은 혼인기간이 7년 이내인 무주택세대구성원에게...
```

### Ⅱ. 질의 답변 → Q&A 쌍 단위 청킹

FAQ 문서와 동일하게 질문 + 답변을 하나의 청크로 묶습니다.

**Q번호 패턴**: `Q장번호-질문번호.` (예: `Q1-1.`, `Q4-12.`)

```
Q1-2. 부(세대주), 모, 자(子), 자의 배우자가 있는 가정에서
       무주택세대구성원이 될 수 있는 사람은 누구인지?

A 부 : 세대주이므로 무주택세대구성원이 될 수 있음
  모, 자 : 세대주의 배우자와 직계비속이므로 무주택세대구성원이 될 수 있음
  자의 배우자 : 세대주의 배우자도 아니고 직계존비속도 아니므로
                무주택세대구성원이 될 수 없음
```

## 5. 전처리 흐름

```
PDF 텍스트 추출 (pdftotext)
        ↓
파트 분리 (Ⅰ. 본문 / Ⅱ. 질의 답변)
        ↓
        ├─ [본문 파트] ─────────────────────────────────────────
        │   목차·표지·파트 구분 페이지 제거
        │           ↓
        │   장·절 단위로 분리
        │           ↓
        │   절 길이 확인
        │   ├─ 2,000자 이하 → 절 통째로 청크
        │   └─ 2,000자 초과 → 소제목(◦, 숫자, 가나다) 단위로 분리
        │           ↓
        │   절 제목을 page_content 앞에 prefix로 포함
        │
        └─ [질의답변 파트] ──────────────────────────────────────
            Q번호 패턴(Q장-번호.) 기준으로 Q&A 쌍 분리
                    ↓
            헤더("주택공급 업무 매뉴얼", "Ⅱ. 질의 답변") 제거
                    ↓
            Q&A 쌍 단위 청크 생성
```

## 6. Metadata 설계

```python

    "source":        "주택공급 업무 매뉴얼",
    "source_year":   "2017",
    "part":          "본문",
    "chapter":       "제4장",
    "chapter_title": "주택공급 방법",
    "section":       "제4절",
    "section_title": "특별공급",
    "subsection":    "신혼부부 특별공급",  # 소제목 분리 시에만
    "chunk_level":   "section",           # 또는 "subsection"
```

## 8. 전체 요약

| 항목 | 설계 내용 |
|------|-----------|
| 파트 분리 | 본문 / 질의 답변 별도 처리 |

| 빈 페이지 제거 | 목차, 표지, 파트 구분 디자인 페이지 |
