# LH 분양가이드 청킹 전략

## 1. 문서 개요

LH청약플러스(apply.lh.or.kr) 웹페이지를 PDF로 캡처한 4종 문서입니다.

| 파일명 | 페이지 | 내용 |
|--------|-------|------|
| 분양가이드___분양절차.pdf | 2페이지 | 아파트 분양 8단계 절차 |
| 분양가이드__신청자격-일반공급pdf.pdf | 3페이지 | 공공분양 일반공급 자격 및 소득·자산기준 |
| LH_청_신청자격-특별공급.pdf | 15페이지 | 특별공급 6종 자격·소득기준·당첨기준 |
| 분양가이드___전매제한.pdf | 2페이지 | 전매제한기간 및 거주의무기간 |

## 2. 공통 전처리

모든 파일에 반복되는 헤더/푸터 노이즈를 제거합니다.

```
제거 대상:
  "26. 6. 8."                     ← 캡처 날짜
  "오후 9:02" (시간)               ← 캡처 시간
  "분양가이드"                      ← 페이지 상단 탭 메뉴
  "공공분양  신혼희망타운"            ← 탭 메뉴
  "소개 분양절차 신청자격-일반공급..." ← 탭 메뉴
  "https://apply.lh.or.kr/..."    ← URL
  "분양가이드 < 분양주택 < 청약 < LH청약플러스" ← 브레드크럼
  "1/15", "2/15" 등               ← 페이지 번호
```

## 3. 파일별 청킹 전략

### (1) 분양가이드___분양절차.pdf

**구조**: STEP01~STEP08, 단계별 설명이 1~2줄

**전략**: 전체를 단일 청크로 저장

분량이 2페이지, 2,111자로 매우 적고, 8단계 흐름이 연결된 하나의 프로세스입니다. 단계를 쪼개면 문맥이 끊기므로 통째로 저장합니다.

```python
page_content = """
아파트 분양절차 (공공분양 LH)

STEP01. 내게 적합한 아파트 찾기
공공분양 : 전용면적 85㎡ 이하의 내집마련을 하실 수 있는 분양 아파트입니다.

STEP02. 입주자저축 가입 및 청약 신청자격 발생
...
STEP08. 입주하기
잔금 납부 후 입주를 하게 됩니다.
"""

metadata = {
    "source":      "LH 분양가이드",
    "page":        "분양절차",
    "chunk_level": "document",
}
```

### (2) 분양가이드__신청자격-일반공급pdf.pdf

**구조**: 입주자격조건 → 입주자선정 방법(3단계) → 순위 기준 → 당첨자 선정기준 → 소득·자산기준 표

**전략**: 3개 청크로 분리

소득기준 표의 숫자들은 pdftotext로 잘 추출되지만 셀이 분리되어 추출됩니다. 표는 원본 그대로 보존하되, 의미 단위로 묶어서 저장합니다.

| 청크 | 내용 | 비고 |
|------|------|------|
| ① 입주자격 및 선정방법 | 무주택세대구성원 조건, 1·2순위 기준, 당첨자 선정기준(순차제) | 텍스트 위주 |
| ② 소득기준 표 | 공공분양/분양전환공공임대 × 가구원수별 월평균소득 100%/200% | 숫자 표 |
| ③ 자산기준 + 출산가구 완화 | 부동산·자동차 기준, 출산가구 10%p/20%p 완화표 | 표+텍스트 |

```python
metadata = {
    "source":        "LH 분양가이드",
    "page":          "신청자격-일반공급",
    "chunk_level":   "section",
    "section_title": "입주자격 및 선정방법",  # 또는 "소득기준", "자산기준"
}
```

### (3) LH_청_신청자격-특별공급.pdf

**구조**: 특별공급 6종 × (입주자격조건 + 소득기준 표 + 자산기준 + 당첨자선정기준)

**전략**: 특별공급 유형 단위 청킹 (1유형 = 1~2개 청크)

각 유형의 자격조건과 당첨기준은 함께 검색될 가능성이 높으므로 같이 묶고, 소득기준 표는 분량이 크면 별도 청크로 분리합니다.

| 특별공급 유형 | 청크 구성 |
|------------|---------|
| 다자녀(2자녀 이상)가구 | ① 자격조건+당첨기준 / ② 소득기준 표 |
| 노부모부양 | ① 자격조건+당첨기준 / ② 소득기준 표 |
| 신혼부부 | ① 자격조건+입주자선정순위+당첨기준 / ② 소득기준 표 |
| 생애최초 | ① 자격조건+당첨기준 / ② 소득기준 표 |
| 신생아 | ① 자격조건+당첨기준 / ② 소득기준 표 |
| 국가유공자 | 단일 청크 (분량 짧음) |
| 기관추천 | 단일 청크 (분량 짧음) |
| 출산가구 기준완화 공통표 | 단일 청크 (공통 적용) |

```python
metadata = {
    "source":        "LH 분양가이드",
    "page":          "신청자격-특별공급",
    "supply_type":   "신혼부부",           # 특별공급 유형
    "chunk_level":   "supply_type",       # 또는 "income_table"
    "section_title": "자격조건 및 당첨기준", # 또는 "소득기준 표"
}
```

**소득기준 표 처리**

pdftotext로 추출하면 숫자가 셀 단위로 분리되어 나옵니다. RAG용으로는 구조화된 텍스트로 재구성해서 저장하는 것이 embedding 품질에 유리합니다.

```
# 재구성 후 저장
[신혼부부 특별공급 소득기준 - 공공분양 우선공급 100%]
3인 이하: 7,533,763원 / 4인: 8,802,202원 / 5인: 9,326,985원 ...
```

**구조**: 전매제한 제도개요 → 전매제한기간 표 → 거주의무기간 표

| 청크 | 내용 |
|------|------|

```python
metadata = {
    "source":        "LH 분양가이드",
    "page":          "전매제한",
    "chunk_level":   "section",
    "section_title": "전매제한기간",  # 또는 "거주의무기간"
}
```

## 4. 전처리 흐름

```
PDF 텍스트 추출 (pdftotext)
        ↓
헤더/푸터 제거
  - 날짜/시간 ("26. 6. 8.", "오후 9:02")
  - 탭 메뉴 ("분양가이드", "공공분양", "신혼희망타운" 등)
  - URL (https://apply.lh.or.kr/...)
  - 브레드크럼 ("분양가이드 < 분양주택 < 청약 < LH청약플러스")
  - 페이지 번호 ("1/15" 등)
        ↓
파일별 청킹
  분양절차   → 단일 청크
  일반공급   → 3개 청크 (자격/소득표/자산)
  특별공급   → 유형별 청크 (총 약 16개)
  전매제한   → 2개 청크
        ↓
ChromaDB 저장
```

## 6. 기존 문서와의 관계

이 4종 문서는 기존 3종(법령, FAQ, 매뉴얼)과 내용이 일부 겹치며, **LH 공공분양에 특화된 최신(2026) 기준**입니다.

| 항목 | LH 분양가이드 | 2024 FAQ | 주택공급 매뉴얼 |
|------|-------------|---------|--------------|
| 기준 연도 | 2026 | 2024 | 2017 |
| 적용 대상 | LH 공공분양 특화 | 전체 청약 일반 | 전체 청약 일반 |
| 소득기준 수치 | 최신 수치 포함 | 수치 미포함 | 수치 미포함 |
| 특별공급 종류 | 신생아 포함 (최신) | 신생아 포함 | 신생아 미포함 |

`source_year="2026"` 으로 설정하면 검색 시 최신 기준으로 필터링 가능합니다.
